# 전원 인가 벤치 절차 (첫 호버 전 안전 관문)

`dual_imu_cascade_pwm` 기준. **손으로 기체를 잡은 채 프로펠러 저속**으로,
비행 없이 물리 신호 체인을 검증한다. 호스트 SIL(commit e6d0908)이 **못
닫은 것**을 여기서 닫는다:

- 실제 IMU의 장착 방향 (기울임 → 올바른 부호의 angle 텔레메트리)
- 실제 GPIO→모터 매핑과 믹서 부호가 **물리 추력**으로 맞는가
- 실제 프로펠러 회전 방향
- **yaw 부호** (SIL S5에서 미해결로 남긴 항목)

SIL은 믹서·제어법칙의 부호 자기일관성만 확인했다. 이 벤치가 실센서·실모터·
실프롭을 검증한다. 둘은 상호보완이다.

---

## 0. 핵심 안전 원칙 (읽고 시작)

1. **저스로틀이 안전장치다.** 모든 부호는 최대 권한이 나와도 추력이 약한
   저스로틀에서 검증한다. 부호가 틀려도 위험한 추력이 안 나오는 구간에서만
   확인한다.
2. **부호 검증은 먼저 프롭 OFF·무추력으로 한다.** Tier1 텔레메트리
   `Motor_M1..M4`가 각 모터 PWM(µs)을 실시간으로 보여주므로, 프롭 없이 손으로
   기울이며 **숫자만 읽어** 전체 부호 체인을 확인할 수 있다. 프롭 ON은 그 숫자
   대로 추력이 나오는지 저속으로 확인하는 단계다.
3. **킬 경로를 프롭 돌리기 전에 검증하고, 손 닿는 곳에 둔다.** `stop` 명령,
   RC 타임아웃(500ms), 그리고 **배터리 물리 분리**(최종 킬). 배터리 커넥터를
   즉시 뽑을 수 있게 준비.
4. **손 위치:** 기체 **프레임(암/센터)**만 단단히 잡는다. 손가락·머리카락·
   옷·케이블을 4개 프로펠러 디스크에서 전부 치운다. 보안경 착용.
5. **역할 분리:** 가능하면 한 명은 기체를 잡고, 한 명은 키보드/패드. 혼자면
   disarm을 한 손에 두고 시작.
6. **틀리면 밀어붙이지 않는다.** 어떤 모터가 "내려가야 하는데 올라간다" →
   **즉시 stop**, 원인 수정, 재검증. 스로틀을 올려 "되게" 만들려 하지 말 것.

전류 제한 전원(랩 PSU, 예: 5A 리밋)이 있으면 폭주 시 피해를 줄인다. 없으면
배터리로 하되 저스로틀 원칙을 더 엄격히.

---

## 1. 준비

- 기체를 안정된 곳(또는 손)에 두고, 프로펠러 **제거** 상태로 시작.
- 텔레메트리 모니터 실행:
  ```bash
  python scripts/monitor_telemetry.py
  ```
  확인할 필드: `Calibration_OK`, `Armed`, `angleX/Y/Z`,
  `Motor_M1..M4`, `PID_Loop_Hz`, `TgtRate_Roll/Pitch/Yaw`, `Fault_*`.
- 모터 물리 위치 라벨(펌웨어 핀 매핑):

  | 텔레메트리 | GPIO | 위치 | 프롭 방향(설계) |
  |---|---|---|---|
  | `Motor_M1` | 4 | **FL** 전-좌 | CW |
  | `Motor_M2` | 5 | **RR** 후-우 | CW |
  | `Motor_M3` | 6 | **FR** 전-우 | CCW |
  | `Motor_M4` | 7 | **RL** 후-좌 | CCW |

- RC 링크(무장 시 500ms 워치독을 먹여야 함): `control_dualsense.py`(게임패드)
  로 arm/disarm + 스틱 중앙=수평 목표(target 0)를 스트리밍하는 것을 권장.
  패드가 없으면 임의 UDP 송신기로 `rc <seq> 0 0 0`를 **≥5Hz**로 흘려보낸다.

## Stage A — 프롭 OFF: ESC 아밍 · 회전 방향 · 매끄러움

**목적:** 각 ESC가 아밍되고 매끄럽게 돌며, 회전 방향이 설계와 맞는지. 회전
방향은 3상 배선으로 정해지므로 PWM 주파수와 무관하다(그래서 50Hz 진단
펌웨어로 확인해도 유효).

1. `motor_pwm_bench` 펌웨어를 플래시. (**주의:** 이 스케치는 부팅 7초 뒤
   자동으로 4모터를 30%까지 동시 램프한다. 프롭 반드시 OFF, 기체 고정.)
2. 각 모터 벨(bell)에 매직/테이프로 방향 표식.
3. 30% 유지 구간에서 **위에서 내려다보며** 4개 방향을 동시에 확인:
   FL(M1)·RR(M2)=**CW**, FR(M3)·RL(M4)=**CCW**.
4. 각 모터가 떨림/역회전/코깅 없이 매끄럽게 도는지, 아밍음이 정상인지.

**방향이 틀리면:** 그 모터의 3상 중 **아무 두 선을 교체**하고 재확인.
**틀린 방향으로 프롭 달고 진행 금지** — 추력이 위로 나거나 yaw가 반대가 된다.

> `motor_pwm_bench`는 개별 모터 제어가 없다. 어느 모터가 M1인지 격리 확인이
> 필요하면, `writeAll`을 한 모터만 쓰도록 3줄 수정한 임시 스케치를 쓰거나,
> Stage B의 폐루프 응답(아래)으로 위치별 매핑을 함께 검증한다.

---

## Stage B — 비행펌웨어, 프롭 OFF: 킬 경로 + **부호 전체(무추력) 검증**

`dual_imu_cascade_pwm`를 플래시. **프롭은 아직 OFF.** ESC가 이 펌웨어의 400Hz
PWM에서도 아밍되는지 먼저 확인(부팅 시 `stopMotors`로 정지 신호 유지).

### B-1. 부팅/상태
- 시리얼에 `[CALIB] OK`. 텔레메트리 흐름 정상, `Armed=0`.
- `gains` 명령 → **Ki_Rate_Roll/Pitch = 0.0500** 인지 확인(방금 조정한 값이
  실제 반영됐는지).

### B-2. 킬 경로 검증 (프롭 돌리기 전에!)
RC 스트림을 켜고 `start`(무장). 저스로틀 `th 1150`(프롭 없으니 안전).
1. **stop 명령:** `stop` → `Motor_M1..M4`가 전부 1000, `Armed=0`.
2. **RC 타임아웃:** 다시 `start` 후 rc 스트림을 **끊는다** → 0.5초 내
   `Fault_RC` 세워지고 모터 1000으로. (복구는 다음 `start`.)
3. **오버틸트:** `start` 후 기체를 60° 이상 기울이면 `Fault_*`(over-tilt)로
   잠기고 모터 1000. (SAFETY_ANGLE=60°)
4. **물리 킬:** 배터리 커넥터 분리로 즉시 정지되는지 손에 익힌다.

### B-3. 부호 체인 검증 — 무추력, 텔레메트리 숫자만 읽기 ★
`start` + `th 1150`, rc 스트림 중앙(target 0). 기체를 손으로 살짝 기울이며
**`Motor_M1..M4` 숫자 변화**를 본다. 관례에 의존하지 않는 **불변식**:

> **누른(아래로 내린) 쪽의 모터가 반드시 올라가야 한다** (그 쪽을 다시 밀어
> 올리려고). 반대로 **든(위로 올라간) 쪽 모터가 올라가면 부호 반전 → 즉시 stop.**

| 손 동작 | 올라가야 하는 모터 | 내려가야 하는 모터 | angle 확인 |
|---|---|---|---|
| **우측을 아래로**(roll) | M2(RR), M3(FR) = 우측 | M1(FL), M4(RL) | `angleX`가 한 방향으로, 수평복귀 시 0 |
| **좌측을 아래로** | M1(FL), M4(RL) = 좌측 | M2(RR), M3(FR) | `angleX` 반대 부호 |
| **기수를 아래로**(pitch) | M1(FL), M3(FR) = 앞 | M2(RR), M4(RL) | `angleY` 한 방향 |
| **꼬리를 아래로** | M2(RR), M4(RL) = 뒤 | M1(FL), M3(FR) | `angleY` 반대 부호 |

- 각 방향에서 **기울인 직후 순간**의 모터 반응을 읽는다(적분기가 쌓이기 전,
  P+D 지배 구간). 검사 사이에 수평으로 되돌려 `th 1050` 등으로 잠깐 낮추면
  적분기(>1100에서만 누적)가 리셋된다.
- `TgtRate_*` 텔레메트리도 함께: 우측 아래로 기울이면 `TgtRate_Roll`이
  복귀 방향으로 명령되는지.

**여기서 roll/pitch 부호 체인 전체(실IMU 방향 포함)를 추력 위험 없이
확정한다.** SIL이 시뮬로 확인한 것을 실하드웨어로 닫는 단계.

**틀리면(든 쪽이 올라감):** stop. 원인 후보 — IMU 장착 방향/축 부호, 믹서 부호,
outer/inner 부호. 텔레메트리 `angle` 부호가 먼저 맞는지 보고(센서 문제 vs 믹서
문제 구분), 수정 후 재검증.

## Stage C — 프롭 ON, 손 파지, 저속: 추력 방향 + yaw 부호

Stage B에서 부호가 **숫자로** 확인됐으므로, 이제 프롭을 달고 **숫자대로 추력이
나오는지** 저속으로 확인한다. 프레임을 단단히 잡고 시작.

### C-1. 추력 방향 (매우 저속)
- 프롭 장착. `start`, `th 1080`부터(겨우 도는 수준). rc 중앙.
- 기체가 손을 **아래로 누르는지**(정상 = 위로 뜨려는 추력). 위로 당기면 프롭
   방향/장착이 반대다 → stop, Stage A로.
- 스로틀은 **손 파지 중 ~1150 이하 유지.** 틸트는 작게.

### C-2. 틸트 복원 (물리)
- 저속 유지 상태에서 Stage B-3의 각 방향으로 **작게** 기울여, 텔레메트리에서
   올라간 모터가 실제로 그 쪽을 **밀어 올리는** 반작용이 손에 느껴지는지.
   숫자(B-3)와 물리(C-2)가 일치해야 한다.

### C-3. yaw 부호 — SIL S5 미해결 항목 닫기 ★
- `yaw 1`로 yaw 유지 활성. 저속 유지.
- 기체를 위에서 볼 때 한 방향으로 **살짝 비틀면**, 컨트롤러가 **그 비틀림에
   저항**(복원)하는 반작용을 내야 정상. **비틀림을 키우면(증폭) yaw 부호 반전.**
   불변식: **"내가 준 비틀림을 거스른다."**
- 증폭이면 stop. 원인 — 믹서 yaw 부호 또는 프롭 회전 방향(Stage A). yaw
   반작용은 프롭 방향에 의존하므로 Stage A가 선행돼야 의미가 있다.

> SIL S5는 CW=+ 관례에서 yaw가 발산했다(리포트만). 실제 부호는 여기서 프롭
> 방향과 함께 확정한다. 관례 추정이 아니라 **실측**이 판정한다.

## Stage D — (옵션) 적분기/트림 관찰

- 저속 손 파지, 한쪽으로 작은 정상 편향을 준 채 몇 초 유지 → `iTerm`이 편향을
   상쇄하는 방향으로 서서히 쌓이고(Ki=0.05가 **활성**), `Motor_*`/`TgtRate_*`가
   **매끄럽게**(진동 없이) 움직이는지.
- 적분기가 진동을 유발하면(±50µs 클램프를 빠르게 왕복, 모터가 떨림) Ki가
   실기엔 큼 → 낮춘다. 반대로 편향을 전혀 못 잡으면 → 올린다. **최종 Ki는 이
   테더 관찰로 튜닝**한다(0.05는 SIL 근거 보수적 시작값).

---

## 중단 기준 (하나라도 해당 시 즉시 stop + 배터리 분리)

- 든 쪽 모터가 올라감(부호 반전) / 프롭이 위로 당김(방향 반전)
- yaw가 비틀림을 증폭
- 모터 떨림·역회전·아밍 실패
- 텔레메트리 끊김/`PID_Loop_Hz` 급락, `Fault_*` 예기치 않게 세워짐
- 기체가 손에서 튀려 함 → 스로틀이 예상보다 높다, 즉시 낮춤/정지

## 통과 기록 체크리스트

- [ ] A: 4모터 아밍·매끄러움·방향(FL/RR=CW, FR/RL=CCW) OK
- [ ] B-1: `[CALIB] OK`, `gains` Ki=0.05 확인
- [ ] B-2: stop / RC타임아웃 / 오버틸트 / 배터리킬 4종 킬 검증
- [ ] B-3: roll·pitch 부호(누른 쪽 상승) + angle 부호 OK (무추력)
- [ ] C-1/2: 추력 아래로 누름 + 틸트 물리 복원 OK
- [ ] C-3: yaw가 비틀림에 저항(복원) OK ← SIL S5 닫힘
- [ ] D: 적분기 활성·비진동 (또는 Ki 조정 기록)

**전부 통과해야 첫 (테더/저고도) 호버 시도 자격.** 하나라도 미확인이면 그 항목이
곧 첫 호버의 사고 지점이다.
